# Convention de commits (FR)

## Format recommande

`type(portee): resume court en francais`

Exemples:

- `feat(home): ajoute la section hero`
- `fix(header): corrige l'alignement des boutons`
- `style(typographie): harmonise la police globale`
- `docs(readme): precise les etapes de deploiement`

## Types conseilles

- `feat`: ajout d'une fonctionnalite
- `fix`: correction de bug
- `style`: modification visuelle / CSS sans logique metier
- `refactor`: restructuration du code sans changement fonctionnel
- `perf`: amelioration des performances
- `docs`: documentation
- `build`: scripts, dependances, configuration build
- `chore`: taches techniques diverses

## Bonnes pratiques

- Utiliser l'imperatif present: "ajoute", "corrige", "ameliore".
- Garder un resume court (idealement < 72 caracteres).
- Faire un commit par objectif coherent.
- Ajouter un corps de message si le changement est important.
